# 記事執筆ルール（架空事象版）

## 目的

Vibepedia の記事は、**実在しない架空の対象**のみを扱う。  
読者体験としては、架空対象が実在する制度・設備・出来事であるかのように記述する。

## 必須ルール

1. 実在対象の直接解説を禁止する
- 実在の人物、団体、国家、史実、学術分野をそのまま説明しない。
- 既存の題名を使う場合も、作中世界の固有制度名・案件名として再定義する。

2. 事典文体で記述する
- 断定調（「〜である」「〜とされる」）で統一する。
- 推測・メタ表現（「架空」「フィクション」「創作」等）を書かない。

3. 実在感を担保する
- 管轄機関名、運用拠点、規程名、改訂年、運用指標などの具体情報を含める。
- 時系列情報（導入年、改訂年、監査年など）を本文または `events` に最低1つ以上含める。

4. 記事同士の類似を避ける
- 各記事で対象の性質・運用背景・観測特性・連携先を変える。
- 同一テンプレートの定型文を連続利用しない。語彙・構成・見出しを分散させる。

5. 関連性は許容する
- 他記事との連携・参照・依存関係は明示してよい。
- ただし、関連があっても各記事が独立して読めるようにする。

6. 可読性を担保する
- 1段落を長くしすぎない。目安として2〜4文で区切る。
- 長い列挙や条件は箇条書き・番号付きリストに分解する。
- 手順・判定基準・運用区分は小見出し（`###`）を使って分離する。
- 専門用語は初出時に短く定義し、同一記事内で呼称をぶらさない。

## フロントマター（現行実装準拠）

以下のキーを使用する（`src/content.config.ts` 準拠）。

```yaml
---
slug: "記事タイトル"
title: "記事タイトル"
description: "対象を1文で定義する。"
date: "2026-02-06T16:33:58.789825+00:00"
lastmod: "2026-02-06T16:33:58.789825+00:00"
featured: false
views: 0
category: "科学"
events:
  - date: "05-09"
    text: "作中世界の運用史イベント。"
---
```

### 各項目の運用

- `title`: 作中世界での固有名として記述する。
- `slug`: `title` と同一の値を設定する。
- `description`: 架空対象の定義文にする（メタ説明は不可）。
- `date`: 記事の基準作成日時。
- `lastmod`: 記事の最終更新日時（更新時に必ず見直す）。
- `featured`: トップの「選り抜き記事」掲載フラグ。
- `views`: 整数値。
- `category`: 日本語カテゴリを使用する。現行値は `科学` / `芸術` / `地理` / `数学` / `文化` / `場所` / `歴史`。
- `events`: `MM-DD` 形式の `date` と `text` を持つ配列。作中世界の制度改訂・監査・運用開始などを記述する。

## ファイル命名

- 記事ファイル名は `title` と同一にする（例: `木曜日の裏側.md`）。

## 本文構成の目安

- 先頭は `## 概要` とし、対象の定義・位置づけを示す。
- 中盤で運用・観測・分類・事例・議論などを扱う。
- 末尾に `## 関連項目` を置き、`[[記事名]]` 形式で内部参照を記載する。
- 長文が続く場合は、見出し追加・段落分割・箇条書き化を優先して可読性を確保する。

## 禁止事項

- 実在の事件・技術史・人物史の引用や転記
- 「これは架空である」と明言する注記
- 複数記事での本文コピペ（固有名のみ差し替える行為を含む）
